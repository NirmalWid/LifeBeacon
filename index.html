About the System

When disaster strikes and traditional networks fail, communication becomes a matter of life and death. Our project delivers a fully decentralized, infrastructure-free communication and localization system designed for post-disaster environments. Using Bluetooth Low Energy (BLE) and Wi-Fi Direct, we create a real-time ad-hoc network between smartphones and microcontrollers - no mobile data or internet required. A built-in self-healing mechanism ensures continuous communication by automatically rerouting data when nodes fail.
The system includes a secure SOS messaging platform for broadcasting or directly sending encrypted emergency alerts, and a GPS-free victim localization feature that uses ESP32 sensors to detect nearby phones and estimate their positions with high accuracy. All data is managed through a hybrid database model, combining offline storage on mobile devices with a centralized Cassandra database for scalable, real-time monitoring through a command dashboard.

This is more than just a communication tool - it’s a lifeline built to work when everything else fails.

Decentralized Ad-Hoc Communication Network

Utilizes BLE and Wi-Fi Direct to form a Mobile Ad-Hoc Network (MANET) between smartphones and microcontrollers without any dependence on cellular or internet infrastructure.

Self-Healing Network Mechanism

Automatically detects disconnected or failed nodes and dynamically reroutes messages through available devices to maintain communication.

Secure SOS Messaging System

Enables victims and responders to send broadcast or targeted SOS messages, fully encrypted and prioritized for low-latency delivery.

GPS-Free Victim Localization System

Detects smartphones by capturing Wi-Fi probe requests using ESP32 devices and estimates locations via RSSI-based trilateration.

Dual-Layer Data Management with Real-Time Dashboard

Integrates SQLite on mobile devices for offline data capture and Apache Cassandra at the base station for scalable storage and analysis.

LITERATURE SURVEY

Technologies like Bluetooth Low Energy (BLE), Wi-Fi Direct, and Mobile Ad-Hoc Networks (MANETs) enable direct device-to-device communication without relying on centralized infrastructure. In disaster scenarios, where mobile networks are often down, these technologies help establish decentralized emergency networks that restore vital connectivity. However, MANETs can be affected by node mobility and environmental interference. To address these challenges, BLE and Wi-Fi Direct have been integrated into disaster communication systems, offering more stable, energy-efficient, and reliable solutions.
BLE stands out for its low power consumption and native support on most smartphones, making it ideal in power-limited environments. Its design supports short, intermittent communication while maintaining network resilience. Self-healing algorithms further improve this by automatically reconfiguring routes when devices disconnect, ensuring uninterrupted communication.
Wi-Fi Direct complements BLE by enabling high-speed, long-range peer-to-peer connections without internet access. Its ability to handle large data transfers quickly makes it essential for fast information sharing during emergencies. Together, BLE and Wi-Fi Direct form a hybrid communication layer that balances efficiency, range, and reliability.
To support emergency communication, ad-hoc SOS messaging systems have been developed that bypass traditional delays and ensure fast, secure message delivery. These systems use end-to-end encryption to protect sensitive information and ensure message confidentiality. Additionally, Wi-Fi probe request analysis enables real-time victim localization. As smartphones search for networks, they emit signals that can be detected by nearby ESP32 devices. By measuring signal strength from multiple detection points, the system can estimate the victim’s location—even without GPS or user interaction.
For data handling, a hybrid model is used: SQLite supports fast, local data storage on mobile devices, while Apache Cassandra provides scalable, fault-tolerant storage at the central server. A real-time dashboard allows rescue coordinators to track messages, monitor device activity, and visualize victim locations, enabling smarter and faster decision-making.
By combining MANETs, BLE, Wi-Fi Direct, passive localization, encrypted messaging, and resilient data management, this system delivers a powerful, scalable solution for disaster response. It ensures continuous communication, precise victim tracking, and robust coordination in even the most infrastructure-deficient environments. users in tropical regions.

RESEARCH PROBLEM

During natural or man-made disasters, traditional communication systems such as cellular networks and internet services frequently collapse. This leaves victims disconnected and rescue teams unable to coordinate effectively. Existing solutions depend heavily on this infrastructure, making them unreliable in real emergency conditions where independent, infrastructure-free communication is essential.

Most current systems fail to offer a complete solution that combines decentralized communication, mobility support, secure SOS messaging, and real-time adaptability. Mobile Ad-Hoc Networks (MANETs), while promising, often suffer from instability due to node mobility and lack dynamic self-healing capabilities. Additionally, many systems lack proper encryption, prioritization of emergency messages, and the flexibility to function fully offline, which is critical during peak crisis periods.

In many disaster environments, GPS signals are weak or blocked, making it difficult to locate victims. Existing location-tracking methods often require user interaction or rely on internet-based services. Moreover, the absence of an integrated platform to visualize real-time victim locations, device health, and message flow significantly limits emergency teams' ability to make informed, timely decisions.

RESEARCH GAP

Despite advancements in disaster communication technologies, there is a notable lack of an integrated, infrastructure-independent system that can ensure reliable communication, secure messaging, and real-time victim localization in disconnected environments. Existing solutions often focus on isolated components such as offline messaging, MANETs, or GPS-based tracking, but fail to combine these into a unified, scalable framework. Most systems also depend on centralized infrastructure, lack self-healing capabilities, and are ineffective in highly dynamic disaster zones where connectivity is unstable and user interaction is limited. Furthermore, current victim localization methods typically rely on GPS or active user input, which is impractical in collapsed buildings or unconscious victim scenarios. This gap highlights the need for a decentralized, self-adaptive solution that enables autonomous communication, passive localization, and centralized monitoring - entirely independent of traditional networks.

MAIN OBJECTIVE

To design and implement a decentralized, disaster-resilient telecommunication system that enables infrastructure-free communication, secure SOS messaging, real-time victim localization, and reliable data management in post-disaster environments using commonly available mobile technologies.


Specific Objectives

To develop a hybrid ad-hoc communication network using Bluetooth Low Energy (BLE) and Wi-Fi Direct for device-to-device messaging without relying on internet or cellular infrastructure.

To implement a secure and prioritized SOS messaging system that supports both broadcast and targeted messages with end-to-end encryption and offline functionality.

To design a self-healing network mechanism that dynamically reroutes communication paths and automatically reintegrates disconnected nodes to maintain continuous connectivity.

To establish a GPS-independent victim localization system using Wi-Fi probe request analysis and RSSI-based trilateration with ESP32 devices.

To integrate a dual-database data management model using SQLite for offline storage on mobile devices and Apache Cassandra for centralized, scalable storage with real-time synchronization.

To build a centralized monitoring dashboard that visualizes network status, SOS alerts, and victim locations for improved situational awareness and coordination by rescue teams.

METHODOLOGY

The development of the disaster-resilient telecommunication system followed a structured methodology consisting of design, implementation, and testing phases. The process began with a detailed analysis of existing communication challenges in post-disaster environments, supported by field reports and academic research.
Based on the identified requirements, a hybrid Mobile Ad-Hoc Network (MANET) architecture was designed using Bluetooth Low Energy (BLE) and Wi-Fi Direct to enable peer-to-peer communication without internet or cellular dependency. A cross-platform mobile application was developed using Flutter, incorporating automated device discovery, encrypted SOS messaging, and real-time message relay capabilities. A self-healing mechanism was integrated to detect node failures and dynamically reroute data through alternate paths, ensuring network resilience. For victim localization, ESP32 microcontrollers were configured to operate in promiscuous mode to passively capture Wi-Fi probe requests. By applying RSSI-based trilateration, the system could estimate victim locations without relying on GPS.
Data management was handled using a dual-database approach: SQLite for offline storage on user devices and Apache Cassandra for centralized, high-availability storage at the base station. A centralized dashboard, built using Node-RED, provided real-time monitoring of devices, SOS messages, and victim locations. The complete system was tested under simulated disaster scenarios to evaluate performance metrics such as connection stability, latency, localization accuracy, and data synchronization reliability.
